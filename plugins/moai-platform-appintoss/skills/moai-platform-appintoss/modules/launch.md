# 출시 가이드 (Launch)

앱인토스 미니앱 출시 준비 및 검수 가이드입니다.

---

## 검수 프로세스

### 4단계 검수

앱인토스 미니앱은 4단계 검수를 거쳐 출시됩니다.

| 단계 | 검수 항목 | 담당 |
|------|----------|------|
| 1. 운영 검수 | 앱 정보, 노출 정보, 서류 확인 | 운영팀 |
| 2. 디자인 검수 | TDS 가이드라인 준수 확인 | 디자인팀 |
| 3. 기능 검수 | 핵심 기능 동작 테스트 | QA팀 |
| 4. 보안 검수 | 보안 취약점 점검 | 보안팀 |

**소요 시간:** 영업일 기준 2-3일

### 검수 상태

| 상태 | 설명 |
|------|------|
| 검수 대기 | 검수 요청됨, 대기 중 |
| 검수 중 | 검수 진행 중 |
| 수정 요청 | 수정 필요 사항 발견 |
| 검수 완료 | 모든 검수 통과 |
| 출시 완료 | 서비스 출시됨 |

## 번들 요구사항

### 기본 요구사항

| 항목 | 요구사항 |
|------|----------|
| 최대 크기 | 100MB (비압축 기준) |
| 형식 | .ait 파일 |
| 구성 | HTML, JS, CSS, 에셋 포함 |

### 번들 생성

```bash
npm run build
```

빌드 완료 후 `.ait` 파일이 생성됩니다.

### 콘솔 업로드

1. 콘솔 접속
2. 미니앱 선택
3. "배포" 탭 이동
4. 번들 파일 업로드
5. 버전 정보 입력
6. 검수 요청

## 게임 앱 체크리스트

게임 카테고리 미니앱의 출시 전 체크리스트입니다.

### 필수 구현 사항

- [ ] **전체 화면 구현**
  - 콘텐츠가 디스플레이 전체를 채움
  - 웹뷰 여백이나 반투명 영역 없음
  - Safe Area 처리 (노치, 카메라 홀, 다이나믹 아일랜드)

- [ ] **네비게이션 바 구현**
  - 닫기 버튼 (X) 필수
  - 더보기 버튼 (...) 필수
  - 플로팅 컴포넌트 형태

- [ ] **사운드 컨트롤**
  - 음소거/음소거 해제 토글
  - 설정 저장 (앱 재시작 시 유지)

- [ ] **접근성 요구사항**
  - 텍스트 대비: 4.5:1 이상
  - 터치 타겟: 최소 44x44pt
  - 색상만으로 정보 전달하지 않음

- [ ] **라이트 모드 전용**
  - 다크 모드 미지원
  - 라이트 모드만 구현

### 기능 체크리스트

- [ ] **연령 등급 표시**
  - 게임 등급 표시 필수
  - 적절한 연령 제한 설정

- [ ] **게임 프로필**
  - 프로필 등록 후 리더보드 접근
  - 프로필 미등록 시 생성 유도

- [ ] **테스트 광고 ID 제거**
  - 프로덕션 광고 ID로 교체
  - 테스트 ID 사용 시 검수 실패

- [ ] **데이터 지속성**
  - 게임 진행 상황 저장
  - 앱 재시작 시 복원

### 금지 사항

- [ ] 다크 모드 구현 금지
- [ ] 서드파티 로그인 금지 (토스 로그인만)
- [ ] 서드파티 결제 금지 (토스페이/인앱결제만)

## 비게임 앱 체크리스트

비게임 카테고리 미니앱의 출시 전 체크리스트입니다.

### 필수 구현 사항

- [ ] **네비게이션 바 구현**
  - 표준 네비게이션 바 사용
  - 뒤로가기, 닫기 기능 필수

- [ ] **응답 시간**
  - 화면 로드: 2초 이내
  - API 응답: 합리적인 시간 내

- [ ] **토스 로그인 전용**
  - 서드파티 로그인 금지
  - 토스 로그인만 사용

- [ ] **접근성 요구사항**
  - 텍스트 대비: 4.5:1 이상
  - 터치 타겟: 최소 44x44pt

### 기능 체크리스트

- [ ] **리스트 기능**
  - 정렬 기능 지원
  - 검색 기능 지원 (해당 시)
  - 필터링 기능 지원 (해당 시)

- [ ] **모든 버튼 동작 확인**
  - 비활성화된 버튼 없음
  - 모든 CTA 버튼 동작 확인

- [ ] **기능성 푸시만 사용**
  - 프로모션 푸시는 게임 앱만 가능
  - 비게임은 기능성 푸시만 허용

## 다크패턴 방지 정책

앱인토스는 사용자 신뢰를 위해 다크패턴을 엄격히 금지합니다.

### 핵심 원칙

> "고객의 예상을 뛰어넘는 디자인은 앱인토스 서비스로 출시될 수 없습니다."

모든 서비스는 별도 학습 없이 직관적으로 사용 가능해야 합니다.

### 5가지 금지 패턴

#### 1. 서비스 진입 시 바텀시트 즉시 표시

**금지:**
- 진입 즉시 전면 광고 바텀시트 표시
- 진입 즉시 알림 동의 요청

**이유:** 예상치 못한 중단은 참여를 방해하고 이탈 위험을 높입니다.

#### 2. 뒤로가기 시 예상치 못한 바텀시트

**금지:**
- 뒤로가기 시 알림 동의 다이얼로그 표시
- 이탈 시 유지 요청 팝업

**이유:** 자율성을 훼손하고 유지 조작처럼 느껴져 신뢰가 하락합니다.

#### 3. 종료 기능 부재

**금지:**
- 파트너 CTA만 선택 가능하고 대안 없음
- 닫기 버튼 없는 팝업

**이유:** 강압적으로 느껴지며 불신과 부정적 감정을 유발합니다.

#### 4. 사용자 플로우 중 예상치 못한 광고

**금지:**
- 아이템 수집 중 전면 광고
- 메뉴 선택 중 갑작스러운 광고

**이유:** 몰입을 방해하고 부정적 브랜드 연상을 만듭니다.

#### 5. 모호하거나 불명확한 CTA 버튼

**금지:**
- 기존 화면 설명을 반복하는 CTA
- 행동 결과가 불분명한 버튼

**이유:** 사용자가 결과 행동에 대해 불확실해지고 전환율이 감소합니다.

### 올바른 예시

| 금지 패턴 | 올바른 패턴 |
|----------|------------|
| 진입 시 즉시 팝업 | 사용자 액션 후 적절한 시점에 요청 |
| 뒤로가기 가로채기 | 정상적인 뒤로가기 동작 허용 |
| CTA만 있는 팝업 | 닫기(X) 버튼 포함 |
| 갑작스러운 광고 | 자연스러운 전환 시점에 광고 |
| "확인하기" (모호) | "포인트 받기" (명확) |

## 공통 체크리스트

### 기술 요구사항

- [ ] **TDS 설치 및 적용**
  - `@toss/tds-mobile` 또는 `@toss/tds-react-native` 사용
  - TDS 컴포넌트 활용

- [ ] **라이트 모드 전용**
  - 다크 모드 구현 안 함
  - 라이트 모드만 지원

- [ ] **핀치 줌 비활성화**
  - 줌 기능 비활성화 필수
  - `user-scalable=no` 설정

- [ ] **번들 크기 100MB 이하**
  - 비압축 기준 100MB 이내
  - 에셋 최적화 필요 시 수행

### 보안 요구사항

- [ ] **보안 취약점 점검 완료**
  - XSS 방지
  - CSRF 방지
  - 민감 정보 암호화

- [ ] **mTLS 인증서 갱신 확인**
  - 유효기간 확인
  - 갱신 일정 관리

- [ ] **민감 정보 노출 금지**
  - API 키 노출 금지
  - 인증서 노출 금지

## 출시 후 관리

### 모니터링

- 에러 발생률 모니터링
- 사용자 피드백 수집
- 성능 지표 추적

### 업데이트

1. 새 번들 빌드
2. 콘솔에 업로드
3. 버전 정보 입력
4. 검수 요청 (간소화된 검수)
5. 출시

### 긴급 대응

- 심각한 버그 발생 시 긴급 패치
- 콘솔에서 긴급 검수 요청 가능
- 보안 이슈 시 즉시 서비스 중단 가능
